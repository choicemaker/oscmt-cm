8 Training

Once a probability model is selected, the training panel displays a table showing the clues contained in that model (Figure 13). Each clue has an ID, a name, a check-box indicating whether this clue is enabled/disabled, a decision, a type, and possibly a report/note modifier. The clue ID corresponds to the position of the clue in the clue-file (i.e. the ClueMaker source). The decision (e.g. `match', `differ', or `hold') and the type ("clue" or "rule") are described in Chapter IV. Additional columns may be displayed based on the machine learning techniques used for last training the model. E.g., Figure 13 shows a weight column for maximum entropy (Chapter VIII.1).

The report modifier column describes whether the name of this clue/rule should be reported to the caller of ChoiceMaker if it fires (this is indicated by report or note in this column) and whether the firing of a clue/rule should force the match record to be returned to the caller, even if the ChoiceMaker decision is differ (report, but not note, forces this). See Chapter IV for more details.

Disabling a clue/rule means that the clue/rule will not be applied to the record pairs. If clue/rule A is disabled and clue/rule B is enabled and references clue/rule A, then the value of the expression of clue/rule A is still evaluated. But clue/rule A will not be directly used for computing the weight.

Initially the Fires, Hits, Misses and %Fires columns of the clue table are blank. These columns are filled with data only after a marked record pair source is selected and the Train, Test, or Train and Test command is chosen from the Model menu or the toolbar. Figure 13 shows an example. The data displayed in the clue table can be sorted by any column by clicking the column header. As in most tables in ModelMaker, columns can also be reordered by dragging them.

The Fires column is only filled in for rules and boolean clues (Chapter IV). The Hits, Misses and %Fires columns are only present if the type of the active clue set is boolean (Chapter IV).

Clicking in any cell in the Fires, Hits, or Misses columns will display the indices of the corresponding record pairs in the pairs list on the right hand side. In Figure 13, the user has clicked in the Misses column of the selected row. The indices of the two pairs on which clue mPatientNamesFirstMiddleLast has misfired are in the pair list. More generally, this sets the marked record pair filter (Section 11).

By checking View->Show Summary Table, additional summary information about the clue performance is displayed between the clue table and the message pane as shown in Figure 14.

Once a model and source have been selected, selecting Model->Train starts the training process of the probability model on the training source. The train dialog asks for a number of values:

  • The firing threshold specifies the minimum number of hits required for a clue to be used in the probability model. If a clue fails to meet this threshold over the marked record pair source, it will automatically be disabled. The default value of 3 is suitable in most cases. The minimum value is 2.
  • Checking Recompile if necessary will automatically rebuild the model before training if the clues or schema file has been changed since the last rebuild.
  • Checking Enable All Clues will re-enable all clues before starting the training process. If it is not checked, only the enabled clues are used. In both cases, clues that don't meet the firing threshold are automatically disabled. The option Enable All Rules works analogous for rules. However, the firing threshold criteria does not apply to rules.
  • The machine learning combo box allows the choice of one of the licensed machine learning techniques. Depending upon this choice, values for different parameters can be set.
  • The chosen machine learning technique may require additional parameters to be set. In the example, maximum entropy requires the maximum number of training iterations to be defined. See Chapter VIII for details.

When training is done the number of Fires, Hits, Misses, and %Fires are displayed for each clue. In addition, data specific to the chosen machine learning technique, e.g., weights for maximum entropy, may be displayed. Unless the model is explicitly saved (see below) no data is stored to disk.

Once a model and source have been selected, selecting Model->Test causes the clues to be evaluated on all the marked record pairs of the testing source. The model is tested by evaluating the match probability on all the record pairs in the source.

Model->Train and Test is equivalent to Model->Train followed by Model->Test, i.e., training on the training source followed by training on the testing source.

Model->Disable All Clues un-checks all the boxes in the Enabled column of the clue table with the result that all clues are disabled.

Model->Enable All Clues checks all the boxes in the Enabled column of the clue table with the result that all clues are enabled.

Model->Reset Weights sets all the clue weights to 1.0.

Model->Save saves the current probability model to disk.
