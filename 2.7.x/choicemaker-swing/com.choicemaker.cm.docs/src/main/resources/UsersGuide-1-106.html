7.3.4 Standard exclusions

The automated blocking algorithm never includes more than one condition on the same database field into a single blocking set. E.g., no blocking set of the form firstName="JIM" and firstName="JAMES" would ever be created.

Furthermore, the automated blocking algorithm never uses the same query field twice in a single blocking field, not even using different stacked values. E.g., suppose that blocking is configured to use firstName to block on both firstName and lastName. Furthermore, assume that the query record has stacked values JIM SMITH and JAMES SMITH. Then this rule says that neither firstName="JIM" and lastName="JIM" nor firstName="JIM" and lastName="JAMES" would ever be created.
