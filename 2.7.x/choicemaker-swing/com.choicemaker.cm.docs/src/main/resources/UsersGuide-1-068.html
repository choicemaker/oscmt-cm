2.5.2 Database reporting

Reporting to the database is currently only available in ChoiceMaker Server J2EE. The extension is com.choicemaker.cm.report.db.dbReporterFactory. The following attributes are supported:

Name

Semantics

Use

Default

dataSourceName

The JNDI name of the data source.

mandatory

N/A

showDetails

Whether an entry for each match should be made in the details table

optional

false

showQueryRecord

Whether the actual data of the query record should be logged. The selection of the fields to log is made in the ChoiceMaker schema (Chapter IX.3).

optional

false

showMatchRecord

Whether the actual data of the match record should be logged. The selection of the fields to log is made in the ChoiceMaker schema Chapter IX.3).

optional

false

When reporting to a database, the data is written into tables TB_CMT_REPORT and TB_CMT_REPORT_DETAILS. Data can be deleted from these tables.

The table TB_CMT_REPORT has the following fields:

Name

Semantics

Format

ID

Sequential ID, primary key.

NUMBER(10)

START_TIME

Time the query was started

TIMESTAMP

DURATION_MS

Total duration of query in milliseconds.

NUMBER

DIFFER_THRESHOLD

Differ threshold, e.g., 0.2.

NUMBER(3, 2)

MATCH_THRESHOLD

Match threshold, e.g., 0.8.

NUMBER(3,2)

MODEL

Name of the probability model.

VARCHAR2(32)

PURPOSE

Value of purpose argument, e.g, batch or interactive.

VARCHAR2(32)

RETURNED_FROM_BLOCKING

Number of records returned from blocking.

NUMBER

QUERY_RECORD

Query record in XML format. Only present if showQueryRecord is true. Only fields set in schema are shown.

XMLType, CLOB in databases without XML support.

PLUGIN_DATA

Data from plugins, e.g., blocking sets.

XMLType, CLOB in databases without XML support.

The table TB_CMT_REPORT_DETAILS has the following fields. Entries are only made if showDetails is true.

Name

Semantics

Format

DETAIL_ID

Sequential ID, primary key.

NUMBER(10)

ID

Foreign key to TB_CMT_REPORT.ID.

NUMBER(10)

DECISION

Match decision, one of D (differ), H (hold), M (match).

CHAR(1)

PROBABILITY

Match probability, e.g., 0.98

NUMBER(3,2)

MATCH_RECORD

Match record in XML format. Only present if showMatchRecord is true. Only fields set in schema are shown.

XMLType, CLOB in databases without XML support.

2.6 Generator

The generator element defines the base directory for the code generated by the generators and the ClueMaker compiler.

2.7 Production probability models

The productionProbabilityModels section lists the probability models to be used by ChoiceMaker servers. The name attribute defines the name under which the model is made available, the file denotes the .model file containing the model. Elements of type property with attributes name and value can be used to specify parameters for plugins. In the example, the name of the database and blocking configurations are listed.
