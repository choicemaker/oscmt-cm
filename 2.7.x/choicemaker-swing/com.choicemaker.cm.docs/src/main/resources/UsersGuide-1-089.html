3 Report

The schema can identify fields whose values are to be stored for each run-time query for later reporting and analysis. Other data, such as the ChoiceMaker probability and decision, can also be stored at run time. The run time report can currently only be written in XML. Support for flat files and databases is forthcoming. This data can be queried to create reports and analyze the performance of ChoiceMaker. Reports can be created with the supplied reporting tools or with any program that can access the report. Furthermore, record data from run time can be viewed in ModelMaker's review screen (Chapter II.10) by using the report file as source data for an XML marked record pair source.

The element reportNodeType has the following attribute:

Name

Semantics

Use

Default

include

The following are legal values:

  • none: no fields of this or nested node types are stored.
  • select: of this node type, only fields that themselves have an attribute report with value true are stored
  • nodeType: all fields of this node type are stored.
  • all: all fields of this and all nested node types are stored.

optional

none

The element recordField has the following attributes:

Name

Semantics

Use

Default

use

Whether the value of this field should be reported. Note that the value of field is only used if include in the enclosing node type's reportNodeType is select and no outer node types' reportNodeType have an include value of none or all.

optional

false
