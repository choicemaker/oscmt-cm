1.3.3 Common elements

Most I/O plugins support the boolean attribute use of xField. If its value is true or the attribute is omitted, the field is present. If its value is false, the field is not present. Line 16 of Figure 66 gives an example of a field that is not present.

If a field is a derived field for a given plugin, it is not read for any configuration of that plugin.

All plugins that perform input also check the value of the optional attribute intern of strings, which itself is a child element of global (Figure 65). If it is true, all Strings assigned to fields are put into their canonical representation via String.intern(). See Chapter III.9 for details.

1.4 ModelMaker GUI

Plugins that support the input or output of marked record pairs may contain a graphical user interface to configure sources and sinks of that type. XML and flat file are examples of this.

Marked record pair sources for ModelMaker consist of two components. The descriptor and the actual source data. The descriptor is a XML file with the extension `.mrps'. It is stored in the directory etc/traindata of the current customization.

Plugins that offer a sink may also allow source data in that format to be generated by storing the contents of other sources or of a selection in that format.

In most cases, the GUI and non-GUI parts are separated into two plugins com.choicemaker.cm.io.x.base and com.choicemaker.cm.io.x.gui.
