7 Marked record pair source handling

ModelMaker works with a training and a testing source. Exactly one of them is active at a time. The single active source is used for all three panels, Train, Test, and Review. The name of the inactive source appears between parentheses in the title bar. Training (Section 8) makes the training source active. Testing (Section 9) switches to the testing source. If there is only a training or only a testing source, this single source is used for all operations.

Two different sources are usually used for training and testing because training and testing on the same source does not give an accurate picture. However, any given source can be used for both training and testing at different times. The two sources can be swapped with Source->Swap Sources. Both sources can be kept in memory by checking Source->Training, which is faster if the computer has enough memory, but can have an adverse effect otherwise.

Whenever the source selection is changed, any data associated with the previous source is automatically removed from the ModelMaker displays.

It is the user's responsibility to choose only sources whose data conform to the schema of the selected model.

7.1 Defining a new marked record pair source

A new source can be defined with Source->Training->New... or Source->Testing->New... This brings up the Source-type Selector Dialog (Figure 11). This dialog allows the user to specify the type of source that is to be defined. The set of available types depends upon the plugins configured in the configuration file (Chapter IX).

Based on the user's selection a Source Builder Dialog of the appropriate type will be displayed. Each such dialog allows the user to specify the file name for the new source. In addition the user may be prompted to supply other parameters specific to the source type.

The description of each source is contained in a separate file. This file contains the name of the source, the type of the source (e.g., XML, flat file, Oracle), and other source type specific parameters. For example, an XML source type descriptor contains the name of the XML file that contains the actual data. Similarly, an Oracle source type descriptor contains the name of the Oracle data source and the SQL query used to retrieve the actual data. The source description only contains a reference to the actual data, but never the actual data.

The source description files are stored in files with extension `.mrps', typically in etc\traindata. The actual pair data may be in this directory or in any other location, including a database.

7.2 Source types

Except for Composite, support for all source types, such as XML, flat file, and database, are provided by I/O plugins. See Chapter IX for their description.
