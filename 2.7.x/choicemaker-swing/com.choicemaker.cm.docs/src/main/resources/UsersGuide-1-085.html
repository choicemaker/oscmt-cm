1.3.1 Multiple configurations

Most I/O plugins support multiple configurations in the same schema. This is necessary if, for example, we want to use two flat files with different field layouts. To support this, the xGlobal, xNodeType, and xField elements of most plugins have a conf attribute. The value of the conf attribute is a comma separated list of configurations to which the element applies. If no conf attribute is present or its value is "all", the element applies to all configurations. If there are multiple xGlobal, xNodeType, or xField elements that apply to a configuration according to this rule, only the first element is considered.

Figure 66 gives an example. On lines 3-5 we define three flat file configurations A, B, and C. Line 9 defines the position and width of firstName for configuration A. Line 10 gives the same information for configuration B. We don't give any explicit configuration information for C, so the defaults apply.

Line 13 defines the position and width of firstName for configurations A and B. Again the defaults apply for configuration C.

Line 16 states that the maidenName field is not present in any configuration.
