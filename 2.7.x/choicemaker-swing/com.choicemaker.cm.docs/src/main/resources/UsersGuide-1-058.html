9 Various

9.1 Exception handling

Like schema expressions (Chapter III.8), clue expressions may use Java methods that have non-empty throw clauses since each clue is wrapped in a separate try-catch block. All exceptions may be logged. Only errors, e.g., OutOfMemoryError, cause program termination.

9.2 Typing of the conditional operator

Unlike Java, ClueMaker gives the smallest possible type to the result of the ternary operator. This means that for booleans b0 and b1, the expression b0 ? 0 : b1 ? 1 : 2 has type byte in ClueMaker rather than int as in Java.
