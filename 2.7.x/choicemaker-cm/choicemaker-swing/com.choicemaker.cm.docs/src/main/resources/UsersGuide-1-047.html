Chapter IV. Clues and ClueMaker

1 Introduction

To compute the probability that two records denote the same physical entity, ChoiceMaker evaluates a number of comparison clues. For example, a clue may detect if the first names of the records compared are the same. Another clue may detect if the phoneticizations of the last names differ. The first clue, if it returns true, would provide evidence that the records denote the same physical entity. The second clue provides evidence to the contrary. The relative importance of evidence embodied by the clues is determined by a machine learning process, such as maximum entropy or support vector machines.

ChoiceMaker has defined a language called ClueMaker to facilitate the writing of clues. ClueMaker is based on a functional subset of Java which should make it easy to learn for Java, C++, and C programmers.

ClueMaker allows us to use any side-effect free Java method from the JDK and third-party libraries in clues. Hence any Java method, including ChoiceMaker's own library of matching methods (Chapter VI) can be used to build clues. Many text editors provide proper automatic indentation for ClueMaker files in the editor's Java mode.
