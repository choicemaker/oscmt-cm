4 Node types, fields, and types

Lines 5 - 15 define the type of the records to be matched. In the example, the nodes to be compared are called person, as stated by the required attribute name of nodeType.

Each person has a firstName, lastName, address, and streetName field. In this example all the fields are of type String, but the type can be any Java type (primitive or reference). The name of a field is defined by the attribute name, the type by the attribute type. Both attributes are required.

The attribute transient governs whether the value of the attribute is transferred to client applications, such as ReviewMaker, that receive record data in ChoiceMaker's internal representation (Section 9). The value is transferred if transient is false. The default value is false.
