6 Derived fields

The field streetName in Figure 31 is a derived field. This means that the value of the field is computed from the value(s) of one or more other fields. In the example, the value is computed by applying the method AddressParser.getStreet to the address field.

The value is only computed if the precondition, given by the optional pre attribute, is true. If the precondition does not hold, the value is null, respectively 0. If the precondition is false, the value of the field is invalid, independent of the validity predicate. If the precondition is omitted, it defaults to true.

A field can be derived or read (non-derived) depending upon the input source. For example, the streetName might be computed for the query record that is input over the call interface, but read if the input source is the database, which may store it for performance improvement, blocking accuracy, or the sake of other applications. The input sources for which the value of the field is to be computed is given by the src attribute. The latter takes a comma separated list of input sources or the special keyword all, as in the example.

Some derived fields are only used for computing other derived fields. For example, a name parser derived field of type ParsedName may be created and used to define the value of standardized first, middle, and last names through the respective getter methods. If the name parser is not used in the clue set, it can be declared to be of local scope (attribute of field scope="nodeInit"), which makes it a local variable that is visible during the computation of the remaining derived fields of the current node only. This can save memory.
