2 ChoiceMaker Server J2EE

ChoiceMaker server is provided as a EAR or a zipped exploded EAR. The EAR contains the following components:

  • ChoiceMakerEjbs.jar: The ChoiceMaker EJBs. The exact contents of this file vary. It is created in Eclipse by merging the set of used plugins into a single jar archive. The contents also vary from J2EE server to J2EE server as some servers contain classes included in ChoiceMaker (e.g., log4j and ant) and do not allow duplicate classes.
  • models.jar: The ChoiceMaker matching models (shared library). This library is created with ModelMaker (Chapter II.12) and includes a set of compiled models along with configuration information. Upon modifications to the model, this library must be regenerated and replaced in the EAR.
  • webdemo.war: Demo Web application showing how to use the ChoiceMaker query service. This module should be removed in production.
  • ReviewMakerSwing.war: The Java Swing ReviewMaker GUI client WebStart archive. This is an optional component.
  • ReviewMakerWeb.war: The Web-based ReviewMaker client. This is an optional component.
It is suggested that customers add all their components (EJB, WAR) to the ChoiceMaker EAR. Alternatively, server-specific functionality may be used to make the class loader of the ChoiceMaker common and model library as well as the EJBs an ancestor of the class loader of the customer's WAR or EAR class loader.
For Java clients, we also provide the following two jars:
  • ChoiceMakerClientLib.jar: The ChoiceMaker EJB client library containing the remote and home interfaces as well as all referenced types.
  • holderClasses.jar: The JavaBeans holder classes for the records of the used models. This library is created with ModelMaker.
    1. EJB API

ChoiceMaker contains two stateless session beans query service and admin service and a stateful session bean review service. Consult the JavaDoc API documentation for details.

Client applications should only interact with these three beans. Additional EJBs are for ChoiceMaker usage only.
