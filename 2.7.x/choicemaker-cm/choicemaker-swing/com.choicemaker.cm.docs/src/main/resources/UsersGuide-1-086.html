1.3.2 Stacking in the root node type

Conceptually, the root node type is never stacked in ChoiceMaker. However, in a database multiple rows in the root table might be linked through a common id. The same holds for flat files and other sources.

This situation is handled by the respective I/O plugins by declaring the root node type to be virtual. All elements of the single directly nested node type are then assembled to form a single entity.
