6 Probability model handling

ModelMaker works with a single model at a time. Upon creating a new model or opening an existing model, that model remains the active model until another model is selected.

An informational message is posted in the message panel indicating that the selected model is now the active model. Additional information about the model's clue file, schema file, and last training run is also posted. This information can also be displayed at any time by choosing Model->Info.

The name of the active model is displayed on the title bar. The clues that this model contains now appear in the training panel. The training panel will be discussed in detail in a later section.

6.1 Creating a new model

A new model can be created by choosing Model->New..., which opens the Model Builder Dialog (Figure 10) that allows the user to specify the model file and the clues file to be used in the model. The location of the clues file can be stored either relative to the model file or as an absolute path. If it is stored relative to the model file, copying both the model and the clues file to a different location works seamlessly. Using an absolute path, the model file by itself can be copied around while the clues file stays in the same directory. A relative path is the default. Note that the choice is governed exclusively by the drop down selector. E.g., if Relative to Model file is selected, the location is stored as a relative path even if the user enters an absolute path, manually or through the file chooser.

Furthermore, the user can choose whether an ant (The Apache Jakarta Project 2002a) target should be called to compile re-loadable Java matching classes (or perform any other operation) upon building the model. The default target, -Dnomain=true compile, compiles the re-loadable Java matching classes.

When the user clicks the Build button, the model is built, added to the list of available models, and stored in the etc\models subdirectory of the ChoiceMaker project.

6.2 Opening an existing model

An existing model can be opened with Model->Open... or by choosing one of the most recently used models from the shortcut list at the end of the Model menu. The properties that were set upon creation of the model can be modified with Model->Edit....

6.3 Rebuilding a model

The probability model needs to be re-built whenever changes are made to the schema or clues. Such changes can be made at any time by directly editing the schema file or clues file with the user's preferred editor. These changes are propagated to the ModelMaker application by selecting Model->Rebuild which compiles the schema and clues files into Java source code and then into Java byte code and finally loads the resulting classes into the running ModelMaker application. This feature allows the user to quickly modify a clue without having to exit and restart the ModelMaker application.

To modify Java classes that are used by the schema or clues, two options for getting the modified version are available:

  1. Exit and then restart ModelMaker.
  2. Put the class or jar files in a place that is not in the classpath of ModelMaker, but on the reload classpath specified in the configuration file (see Chapter V.2.2). In this case, the newly changed Java class and jar files will be used after rebuilding the model. Note that you need to rebuild the model to force loading of the changed Java source code even if you have not modified the schema or clues files. By default, source code in the matching branches gets compiled to classes/matching, which should be on the reload classpath.
6.4 Deleting generated files

After removing plugins or upgrading ChoiceMaker, it may become necessary to delete generated files if an existing model can no longer be loaded. All files generated from the schema and clues files are deleted by selecting Model->Delete generated files.
