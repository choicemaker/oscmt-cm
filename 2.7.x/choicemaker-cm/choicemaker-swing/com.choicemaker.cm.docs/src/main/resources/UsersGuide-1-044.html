10 XML Schemas to define ChoiceMaker schemas

This section deals with the meta description of ChoiceMaker schemas and may be skipped on a first reading.

ChoiceMaker schemas contain metadata about the data to be matched. ChoiceMaker uses XML Schemas to define the structure of ChoiceMaker schemas. Thus, XML Schemas are used as `meta meta' descriptions.

XML Schemas define the structure, content and semantics of XML documents, such as ChoiceMaker schemas. Standard tools can validate XML documents against XML Schemas.

ChoiceMaker has defined an XML Schema for basic ChoiceMaker schemas. Rather than including the specific tags for certain translator generators, this schema contains extension hooks in the form of abstract elements.

The creator of a translator generator defines an XML Schema with the specific elements needed for the particular generator. These elements are defined as extensions of the abstract elements in the base XML Schema.

To deploy ChoiceMaker, developers create a new XML Schema that includes the base Schema along with the Schemas for the translators needed. The base XML Schema is ChoiceMaker/core/etc/meta/ChoiceMaker.xsd. The plugins and customizations have their XML Schemas in etc/meta. It is suggested to use `/' rather than `\' as separator in paths to guarantee compatibility with the widest range of systems (Windows and Unix).

Thanks to the XML Schemas, many XML editors can provide auto completion and other editing aids for ChoiceMaker schemas.
