2 Startup

When ModelMaker is launched, the ModelMaker launchpad (Figure 5) is displayed. Here the user chooses the configuration file. Typically, this is ChoiceMaker\projects\<project>\project.xml. By default, the last used configuration file is pre-selected. Configuration files are discussed in detail in Chapter V.

The launchpad can be bypassed by giving the configuration file with the command line parameter -conf <configuration file name>. For debugging purposes, a log4j logging configuration from the configuration file can be selected with the command line parameter -log <log configuration name>. This is, for example, useful for logging exceptions thrown by clues or expressions in the ChoiceMaker schema. By default, no logging is performed.
