Appendix F. Glossary

ABA Automated blocking algorithm. See blocking.

active clue A clue that fires.

ant A build tool from the Apache Jakarta Project.

approximate database search Search a database for records similar to a query record entered online, find a set of records in a target database which denote the same physical entity (e.g., a person or company) as the query record.

blocking The first pass of record matching. The goal is to quickly find all the records that match the query record and at the same time not return too many records.

blocking condition In blocking, a logical operation that searches for the value of a query field in a database field.

blocking field In blocking, a combination of a query and database field to be used by blocking.

blocking procedure The program that does blocking.

blocking query A database query that is a disjunction (connected by ORs) of some blocking sets.

blocking set A conjunction (connected by ANDs) of some blocking conditions.

ChoiceMaker Server The runtime matching engine.

class loader In Java, an instance of a class that finds and loads classes.

classification A machine learning techniques in which the ML outputs a decision differ, hold, or match.

classpath A list of directories, jar, and zip files in which Java looks for classes referenced by the schema, ClueMaker files, plugins, and classes loaded by them.

clue A small piece of code that compares two records and indicates whether two records match, differ or should be held. Used to compute the probabilistic decision. Written in ClueMaker.

clue file A file used by a model, which contains the clues, i.e., the ClueMaker source, and has the extension `.clues'.

clue set A set of clues and rules in a clue file that can be used in a probability model.

ClueMaker ChoiceMaker's Java-based programming language for writing clues for record comparison.

composite source A combination of one or more other data sources.

confusion matrix A matrix that compares ChoiceMaker's answers with correct (human) answers.

CORBA Common object request broker architecture, a standard architecture for remote procedure calls.

counts In blocking, a table, cached in memory, that counts the occurrences of frequent values. See Chapter IX.7.7.

decision What a clue indicates: match, hold or differ. Certain machine learning techniques, such as maximum entropy, require a decision for each clue.

derived field A field whose value is computed from the value(s) of one or more other fields.

differ threshold For regression-based machine learning techniques, a probability (set by configuration) below which records differ.

duplicate record linkage The same person, business, or thing is present more than once in a database. Duplicate records are linked together.

evaluate Test a model, by using a trained model to match all pairs in a test marked record pair source, and then comparing the human review markings with ChoiceMaker's decisions.

façade The interface to the ChoiceMaker Server.

filter In ModelMaker, a filter can be used to select a subset of a marked record pair source for display in the human review panel.

fire A given clue is said to fire on a record pair when all the conditions for the application of that clue on the pair are met, e.g., a clue that predicted match when the "first name" fields on two records were the same would fire when both "first name" fields were valid and they had the same value.

flat file A plain text file containing record data. Fields separations indicated by delimiters or fixed width boundaries.

group In blocking, a set of database fields that must be all match on a row for blocking to return the record. Only applies to stacked tables. See Chapter IX.7.3.6.

hit A hit occurs when a boolean clue with a decision fires on a record pair that is human-marked obtains the same result as the person.

holder class An automatically generated Java class which is based upon the schema and used to represent nodes in memory.

holds Record pairs that have been marked hold.

human review Review of record pairs for which ChoiceMaker made a hold decision by a human.

jar file Java archive file.

Java byte code Java virtual machine instructions output by a Java compiler that can be executed by the Java virtual machine runtime.

launchpad Start screen from which a user selects the ModelMaker configuration file.

layout The appearance of the human review panel.

long notation The fully expressive ClueMaker form.

machine learning (ML) An artificial intelligence technique to learn the human decision making process from examples.

marked record pair sink A data store into which marked record pairs are output.

marked record pair source data A file with a set of record pairs from the database, reviewed by one or more people and marked match, hold, or differ to indicate whether the constituents denote the same physical entity or not. Typically, 1,000 to 10,000 pairs have to be scored.

marked record pair source description file An XML file that describes marked record pair source data file(s). The description of a data source, with a reference to the actual data, but never the actual data. Stored in files with extension `.mrps'

match record A possible match record that is matched against a query record.

match threshold For regression-based machine learning techniques, a probability (set by configuration) above which records match.

maximum entropy A regression-based machine learning technique.

miss A miss occurs when a boolean clue with a decision fires on a record pair that is human-marked does not obtain the same result as the person.

model A trained set of clues and rules which, hopefully, model the matching in a database.

ModelMaker ChoiceMaker's integrated development environment for the customization of record matching to a particular database. ModelMaker is used for creating and testing probability models.

mrps The extension for a marked record pair source descriptor file.

multi-clue A set of multiple clues that differ from each other by one or more parameters.

nested node A node that is nested within a higher, enclosing, node in the tree structure of nodes of a record.

node An element of a record.

node type The type of a node.

note A ClueMaker modifier that can be placed before the keywords clue and rule (Chapter IV.6). The name and or the number of the clue will then be reported back to the caller by the ChoiceMaker façade.

outer node A node that encloses a nested node in the tree structure of nodes of a record.

plugin A configurable component that provides support for a machine learning technique or a input/output format.

probability model See model.

query record The input to search and blocking.

record A representation of a physical entity, e.g., a person. What ChoiceMaker matches. May contain stacked values, such as multiple names. A record is structured as a tree of nodes.

record pair sink A data store into which record pairs are output.

record pair source An input file containing record pairs.

record sink A data store into which records are output.

regression A machine learning techniques in which the ML outputs a match probability between 0 and 100%. The user can then define two thresholds that determine the decision.

remote method invocation A Java technique for calling a method in another JVM, perhaps on another computer, known as RMI.

report A ClueMaker modifier that can be placed before the keywords clue and rule (Chapter IV.6). The name and or the number of the clue will then be reported back to the caller by the ChoiceMaker façade, even if the decision is "differ".

RMI See remote method invocation.

root node The node at the root of the node tree in a record.

rule The encoding of a business rule in ClueMaker used to override the probabilistic decision for special cases.

schema ChoiceMaker uses an XML-based schema, similar to a database schema, to define the type and structure of the entities to be matched. The schema also defines many other aspects.

schema file A file used by a model, which contains the schema, and has the extension `.schema'.

scoring The second pass of matching. Each potentially matching record returned by blocking is compared individually against the query record. For each record, a decision of differ, hold, or match is made.

shorthand forms A few common clue patterns for which ClueMaker introduces a notational shorthand form.

sink A data store into which data is output, such as a record sink, or a record pair sink, or a marked record pair sink.

source A data source (input).

source field In blocking, a field in the query record.

stacked data Multiple values for a certain field. For example, a person might be represented in a database by one row in a Person table and by multiple rows in an Address table. The Address table is `stacked'.

support vector machines An advanced machine learning technique that supports both classification and regression.

train Train a model. Determine the weights for the clues.

training An automatic machine learning process in which the software `learns', from marked record pairs, the relative importance of clues. The details depend upon the chosen ML technique.

translator class An automatically generated Java class which is based upon the schema and used to exchange record data in a wide variety of formats, such as databases, XML, flat file, and CORBA.

validity conditions Conditions required for the application of the particular clue; these conditions usually includes requirements that the fields contain valid data. See validity predicate.

validity predicate Conditions required for a field in the schema to be valid, defined as a Boolean ClueMaker expression.

weight The relative importance of a clue in maximum entropy. A multiplicative value, so a clue with a weight of 1 has no effect, and a clue with a weight of 6 is as important as 2 clues with weights 2 and 3.

working directory Used to resolve relative file names.

XML A simple, very flexible markup language. It is used for ChoiceMaker schemas, configuration files, report files, and record sources.

XML schemas Documents that define the structure, content and semantics of XML documents.
