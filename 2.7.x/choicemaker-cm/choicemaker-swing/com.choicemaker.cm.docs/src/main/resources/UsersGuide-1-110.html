7.5.2 Adding blocking fields

To add a blocking field f, perform the following steps:

  1. Add the field f to the schema (Section 7.4).
  2. Recompile the model (Chapter II.6.3).
  3. Create one or more indices that have the new field as the first component. Assuming that id denotes the main id of entities, add a composite index on (f, id). Such a covering composite index can answer a blocking query of the form "select id from t where f = `some value'" without requiring a costly access to the table t. If blocking sets that combine f with another field f2 of the same table in the same group are common, add a composite index on (f, f2, id) instead.
