Chapter I. Introduction

1 Record matching

Record-matching tasks can be broken down into three main categories:

  • Duplicate record removal/linkage: The same person, business, or thing is present more than once in a database. Duplicate records are removed or linked together.
  • Database linkage: Two databases are being linked or merged. This might occur, for instance, because of a corporate merger, for purposes of building a data warehouse, or to prevent duplication of effort by storing information common to multiple databases in a single enterprise-wide database.
  • Approximate database search: The ChoiceMaker system can search a database for records similar to a record entered online.

In every one of these cases, the basic problem is essentially the same. Given an input or query record, we are trying to find a set of records in a target database which denote the same physical entity (e.g., a person or company) as the query record.

1.1 Two-pass matching

Figure 1 illustrates ChoiceMaker's two-pass matching:

  1. Query record. A query record is sent to ChoiceMaker. ChoiceMaker's objective is to find all records which might match the query record. The query record might be hand-entered by the user, it might be a record from another database for which a match is sought on the target database, or it could be a record from the target database for which we are seeking duplicate records.
  2. Blocking. ChoiceMaker searches the target database for records which are possible matches to the query record. ChoiceMaker incorporates a proprietary blocking technology which dynamically builds sophisticated queries of the target database to gather a list of possible matches to the query record. The objective at this stage is to retrieve all possible matches and not overly many non-matches. Blocking is described in more detail in Section 2 and in Chapter IX.7.
  3. Many potential matches. This is the set of records returned by blocking as potential matches of the query records.
  4. Scoring. ChoiceMaker's scoring engine determines for each potential match whether it actually denotes or possibly denotes the same physical entity as the query record. Scoring employs a combination of a probabilistic decision computed using a machine learning technique and absolute rules to determine the final decision. Scoring is described in more detail in Section 3 and in Chapter IV and Chapter VIII.
