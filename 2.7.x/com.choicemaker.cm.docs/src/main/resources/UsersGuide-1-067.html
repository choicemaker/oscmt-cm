2.5.1 XML file reporting

XML file reporting writes the report to a XML file. The fileReporter element configures file-based reporting. Its attributes are listed below.

For ChoiceMaker Server J2EE, simply use the corresponding properties. The extensions are:

  • com.choicemaker.cm.report.file.fileReporter for the non-rolling file reporter.
  • com.choicemaker.cm.report.file.periodicRollingFileReporter for the reporter that starts a new report based on a date pattern.
  • com.choicemaker.cm.report.file.rollingFileReporter for the reporter that starts a new report once the current one has reached a certain size.

Name

Semantics

Use

Default

fileName

The name of the file to write to.

mandatory

N/A

type

The name of the reporter, if multiple are configured.

optional

N/A

maxSize

The maximum file size for rolling reports. A number in bytes without suffix, in KB with a `K' suffix, in MB with a `M' suffix or in GB with a `G' suffix. The number may be a floating point.

The report file name is a concatenation of the fileName and a number. For example, if fileName is Report.xml and maxSize is 2M, then first a file Report-0.xml is created. Once this is 2M, a new file Report-1.xml is created.

Note that unlike rolling logs in many systems, old report files are never overwritten if append is true.

A report file may be a few bytes larger, so if for example 2GB is a hard limit, you should use 1.9G as parameter value.

This attribute is only available for the rolling file reporter.

optional, not to be combined with datePattern

N/A

datePattern

A date pattern for periodically rolling logs. The rolling schedule is specified by this option. This pattern should follow the SimpleDateFormat conventions. In particular, you must escape literal text within a pair of single quotes. A formatted version of the date pattern is used as the suffix for the rolled file name.

For example, if the fileName is set to Report.xml and the datePattern set to '.'yyyy-MM-dd, on 2003-02-16 at midnight, a new report file Report.2003-02-16.xml will be created and reporting will continue to this file until it rolls over the next day.

It is possible to specify monthly, weekly, half-daily, daily, hourly, or minutely rollover schedules. See below for details.

Do not use the colon ":" character in anywhere in the datePattern option. It is illegal on some file systems and the text before the colon is interpeted as the protocol specificaion of a URL.

This attribute is only available for the periodic rolling file reporter.

optional, not to be combined with maxSize

N/A

append

Whether a new file should be created each time ChoiceMaker server is restarted or not.

optional

false

showQueryRecord

Whether the actual data of the query record should be logged. The selection of the fields to log is made in the ChoiceMaker schema (Chapter IX.3).

optional

false

showMatchRecord

Whether the actual data of the match record should be logged. The selection of the fields to log is made in the ChoiceMaker schema Chapter IX.3).

optional

false

showClues

Whether the numbers of the firing clues should be logged.

optional

false

showRules

Whether the numbers of the firing rules should be logged.

optional

false

The following lists the possible rollover schedules for datePattern:

datePattern

Rollover schedule

'.'yyyy-MM

Rollover at the beginning of each month.

'.'yyyy-ww

Rollover at the first day of each week. The first day of the week depends on the locale.

'.'yyyy-MM-dd

Rollover at midnight each day.

'.'yyyy-MM-dd-a

Rollover at midnight and midday of each day.

'.'yyyy-MM-dd-HH

Rollover at the top of every hour.

'.'yyyy-MM-dd-HH-mm

Rollover at the beginning of every minute.

If rollover by size or time is used, the created report file is immediately given the numbered or dated name rather than the base name and then be renamed later.
