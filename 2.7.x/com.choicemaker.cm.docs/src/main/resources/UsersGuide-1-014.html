3.1.2.3 Classification vs. regression

There are two varieties of machine learning techniques: classification and regression. In the case of classification, the machine learning technique outputs a decision differ, hold, or match. In the case of regression, the output is a match probability between 0 and 100%.1 The user can then define two thresholds that determine the decision. Records with match probabilities between 0 and the differ threshold, e.g., 40%, are assigned a differ decision. Match probabilities between the differ and match threshold lead to a hold decision, and values above the match threshold to a match decision. This is illustrated in Figure 4.

The values of these thresholds determine the tradeoff between human review (recall) and precision. Since there are two thresholds, the accepted error rate can be set individually for false match and false differ decisions. ModelMaker provides various statistics and graphs to help determine the values of these thresholds.

ChoiceMaker supports multiple machine learning techniques, including maximum entropy and support vector machines. It is described in Chapter II and Chapter VIII.

3.2 Rules

The probabilistic decision can be overridden in certain special constellations for which there exists a clear business rule. For example, if we absolutely trust the Social Security Number (SSN) field, where present, we could use a rule that changes any probabilistic decision to differ if the two records have different SSNs.

Likewise, if we are highly, but not absolutely, confident in the correctness of the SSNs, we could use a rule that changes any match decision between two records that have different SSNs to a hold and leaves all other decisions untouched.

1Depending upon the machine learning technique used these numbers are true probabilities or simply a score normalized to 0..1.
