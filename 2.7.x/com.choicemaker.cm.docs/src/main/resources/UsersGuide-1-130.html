2 ChoiceMaker schemas

This section details the limitations of the current implementation of the generators compiler:

  1. Validity predicates, preconditions of derived fields, and value function of derived fields have to be written in Java rather than ClueMaker. These Java expressions are copied verbatim, except for replacing expressions of the form `valid(fn)', into the generated code. Any errors in them will be signaled by the Java compiler.
  2. In the current version of ChoiceMaker the import statements used in the schema file must also appear in the clues file.
  3. Invalid import statements are flagged by the Java compiler on the generated classes rather than by the generator on the schema.
  4. Some field and record names may lead to clashes with compiler and generator generated identifiers. Most clashes can, however, be avoided by not using double underscores in identifier names.
