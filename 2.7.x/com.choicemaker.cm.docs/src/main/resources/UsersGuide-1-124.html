6 The Review View

The Review view is where the user makes decisions on whether pairs of records match, differ, or should be held for further review. As discussed above, it can be reached from the Overview or Search views by double-clicking on a record or clicking "Review Selection" or it can be reached directly from the menu bar by choosing View->Overview.

shows an example Review screen. In this example, ChoiceMaker has retrieved two records from the database that match the queued record, "Michael Ancrum, born 5/4/1996". The Review screen always displays the queued record in blue at the top of each table, while the matching record from the database is on the bottom in blue (in ClueMaker the queued record would be denoted as the q record, while the database record would be m). Clicking on the tabs labeled "A", "B", etc. enables the user to switch between the different m records. Make your decision on whether the records match or not by clicking the Match, Hold, or Differ radio buttons.

The lower part of the screen is where the data is displayed. This section of the screen displays all the data in a very flexible and intuitive interface that is completely under the user's control. The user can set up the different nodes in the database in multiple tables, showing the fields he/she wants, can set the column widths as desired, and can set up multiple tabbed views as shown in where one tab displays "raw" input data and a second tab shows derived fields, such as, perhaps, normalized addresses. Further details on setting up and viewing data layouts can be found in Chapter II. This data section of the screen works exactly the same as the ModelMaker functionality discussed in that section except that, by default the layout is not editable. Before making any changes to the layout (inserting new tables, widening columns, etc.) you need to click on LayoutàReview/SearchàLayout Editable.

Figure 78 shows the same screen after the user has clicked on the "B" tab. Note that the Review screen requires you to click on every match before any decisions can be accepted, so the "Accept Review" button is now no longer grayed out.

The buttons at the top of the screen perform the obvious functions. Accept Review accepts the current decisions and sends them to the ChoiceMaker server to trigger the corresponding actions (for instance a Match decision would likely result in the two records being linked). Accept Review also removes the record from the queue, although it may remain visible in Overview until Refresh on that screen is clicked. Skip Case simply skips over the current queued record and goes to the next one in the queue. Reset Case sets all decisions to the decisions computed by the ChoiceMaker decision-engine. These are the decisions which are displayed when a record is first brought up in Review. Get Next Record From shows which queue Review is currently retrieving records from, the order in which they are currently sorted in Overview, the records retrieved by Search, or the default Queue. Get Next Record From allows you to switch from one queue to another by clicking on the queue box.
