3.1.2.1 Training

Figure 3 shows a diagram of the training routine for ChoiceMaker. The main steps are:

  1. Design. In this stage, the customization team studies the database and interviews data quality personnel to determine an appropriate set of record matching clues for the problem. It is in this stage that one might discover idiosyncrasies of the database. For instance, a database might be set up to indicate a foreign address by entering 99 in the zip code field.
  2. Clues. The outcome of the design phase is a set of hand-coded clues.
  3. Marked Record Pairs. A set of record pairs is selected from the database using ChoiceMaker's tools. The customer decides for each record pair whether the constituents denote the same physical entity or not. Typically, 1,000 to 10,000 pairs have to be scored. This amounts to as little as 25 hours of clerical time.
  4. Train. Training is an automatic process, the details of which depend upon the chosen machine learning technique. Some techniques, such as maximum entropy, determine the relative importance, or weight, that should be assigned to each clue. Higher weighted clues, like the "match of very rare first names" clue will get a higher weight (i.e., they will predict an overall "match" decision more strongly) than lesser-weighted clues like a "match of common first names" clue. Other techniques learn which combinations of clues indicate a match or differ.
  5. Trained model. The output of training is a trained model.
  6. Test Marked Record Pairs. A portion of the marked record pairs are held back to be used in testing.
  7. Test. The trained system is executed against the test pairs.
  8. Accuracy Okay? If the accuracy is sufficient, the system is ready to be deployed in production. If not, we go back to the design phase to determine what additional clues are needed to improve system accuracy.
