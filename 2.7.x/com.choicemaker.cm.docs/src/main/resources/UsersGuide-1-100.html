5.2 ModelMaker GUI

Figure 66 shows the flat file marked record pair source configuration dialog, which appears when a user wishes to modify or create a flat file marked record pair source. The Source name defines the name of the source as it will appear on the Source->Source menu. It may only contain letters, numbers, spaces, and underscores.

The Source file denotes the file containing the record data. The Browse... button brings up a file chooser to set the file name.

In case of multi-file data, it must be set to the name without the main part. E.g., if the data is stored in Testdescriptor.txt, Testperson.txt, and Testcontact.txt, Test.txt should be entered.

The Multi file checkbox defines whether the data is contained in a single or multiple files. The Single line checkbox defines whether the data is stored on a single line per record pair or not. The Fixed width checkbox defines whether the data is fixed width or delimited. The Delimiter defines the delimiter for delimited files. It can be chosen from the dropdown list or entered in the field on the right after selecting Other: from the dropdown list.

The Tagged checkbox defines whether each line is tagged by the node type. Single-file stacked data must be tagged. This is not currently enforced by the GUI.

The location of the flat file(s) can be stored either relative to the descriptor file or as an absolute path. If it is stored relative to the descriptor file, copying both the descriptor and the flat files to a different location works seamlessly. Using an absolute path, the descriptor file by itself can be copied around while the flat file(s) stays in the same directory. A relative path is the default. Note that the choice is governed exclusively by the drop down selector. E.g., if Relative to Descriptor is selected, the location is stored as a relative path even if the user enters an absolute path, manually or through the file chooser.

Pressing the OK button creates a new source descriptor or modifies an existing descriptor. If a source descriptor of this name already exists, a warning is displayed and the user can cancel the operation. The Cancel button closes the dialog without enacting any changes.

The flat file plugin supports the generation of flat file source data from other sources, like the XML plugin. All of the above parameters must be filled in and a model must be selected to use the source generation. From the Source list, one or more sources must be selected. Their contents are inserted in the list order into the new source(s).

The Remove delimiter from source data checkbox controls whether the delimiter is removed if it occurs within the source data. E.g., if the delimiter is set to "," and one record has a name of "Smith, JR", the comma would be removed from the name field and "Smith JR" would be stored. It is strongly suggested that this option be always used. Otherwise, the field positions might be confused when the file is read again. The option has no effect on fixed-width files.

See Section 4.3 for a description of the round robin and maximum size distribution options.

The GUI for flat file record sources is analogous, except that it does not currently allow the generation of source data.
