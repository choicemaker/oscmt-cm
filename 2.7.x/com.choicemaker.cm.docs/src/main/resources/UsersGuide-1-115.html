8.1.1 Data sources

Oracle data sources (javax.sql.DataSource) can be defined in the configuration file. They are then registered in com.choicemaker.cm.io.db.DataSources. The OracleConnectionCacheImpl from the standard Oracle JDBC drivers is used. Alternatively, the standard pooled connections from the db plugin can be used.

A Oracle data source is configured with a OraConnectionCache element under the oracle element, which itself is a child of plugin. The OraConnectionCache element has a mandatory attribute name and several mandatory children:

Name

Semantics

driverType

Defines the driver type:

thin: the type 4 driver (usually better)

oci8: the type 1 driver

serverName

The IP address or DNS name of the database server. E.g., oracleserver.mycompany.com

networkProtocol

The network protocol. E.g., tcp

databaseName

The name of the database instance. E.g., myinstance

portNumber

The port number. E.g., 1521

user

The user name. E.g., scott

password

The password. E.g., tiger

connectionLimit

The maximum number or connections. The fixed wait scheme is used.

Data sources stored in JNDI can also be used.

8.2 ModelMaker GUI

Figure 71 shows the Oracle marked record pair source configuration dialog, which appears when a user wishes to modify or create an Oracle marked record pair source. The Source name defines the name of the source as it will appear on the Source->Source menu. It may only contain letters, numbers, spaces, and underscores.

The configuration selects the configuration from the ChoiceMaker schema. The Data source dropdown list is used to select the data source, i.e., the connection factory. Under the Select-from-where a query that returns pairs of ids must be entered. This allows any criteria on the pair or the individual records to be used.
