5 Flat files

Flat files are delimited, e.g., CSV or pipe-delimited, and fixed-width plain text files. Support for flat files is provided by com.choicemaker.cm.io.flatfile.base and com.choicemaker.cm.io.flatfile.gui.

Flat files can use any character set supported by the Java implementation on the target platform. This typically includes all common 8- and 16-bit character sets. Dates must be in the format mm/dd/yyyy, e.g., 4/18/2002 stands for April 18, 2002.

The syntax for the ChoiceMaker schema extension is given by etc/meta/Flatfile.xsd.

5.1 Format and configuration

The flat file plugin supports a wide variety of formats. The configuration of the format is split between the ChoiceMaker schema and the source or sink. The schema contains detail information about individual node types and fields, such as the field width. The sources and sinks contain global information, such as whether the file is delimited or fixed width. The sources can be configured through the ModelMaker GUI.

The standard flat file plugin support any combinations of the following:

  • Single or multi file. In the multi-file format, the data of each node type and the pair descriptor1 are in separate files. In the multi-file format, the order of the node types must be the same as of the pair descriptors.
  • Delimited or fixed width. Any delimiter except for carriage return and line feed can be used. In the delimited format, omitted fields are assumed to be null. The fixed width format requires all fields to be present in their full lengths.
  • Tagged or untagged. In tagged format, each line starts with a tag that denotes the node type. Single file data with multiple node types must be tagged.
  • One line per pair or one line per node type format. One line per pair is only supported for non-stacked, non-tagged data in a single file.

For marked record pair sources, the pair descriptor contains the human decision (differ, hold, match), the date when the decision was made, the name of the user who made the decision, the source of the pair, and a comment.

The following is an example of a tagged, single-file, pipe-delimited pair descriptor:

MP|differ|11/27/2001|carolina|cdc|data modified by Andrew

Record sources do not have pair descriptors.
