4 XML

The XML plugins com.choicemaker.cm.io.xml.base and com.choicemaker.cm.io.xml.gui allow records and marked record pairs to be read and written in XML format. XML is well suited if the schema is undergoing change, i.e., fields and records are added and removed. This is because each record and field is identified by its name rather than its position in the source file. On the other hand, XML files are typically 50% slower to process than flat files.

The XML plugin supports all IANA and MIME encodings supported by Xerces.1 Dates must be in the format mm/dd/yyyy, e.g., 4/18/2002 stands for April 18, 2002.

4.1 Format and configuration

Each node type in the schema corresponds to an XML element. Each field corresponds to an attribute. The elements must have the same names as the node types and the attributes the same names as the fields. If the data is in any other format, e.g., elements and attributes of different names or with nested elements in place of attributes, it must first be converted to this format.

Hence, the following is a legal record for the schema StackedSchema.schema (Figure 32):

<person personId="21" firstName="JIM" lastName="SMITH">

<addr address="41 E 11th Street" lastDateReported="01/23/2003"/>

<addr address="51 E 11th"/>

</person>

Note that the presence of all attributes is optional, as illustrated by the missing lastDateReported in the second address.
