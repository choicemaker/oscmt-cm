11 Summary of compilation and training process

Figure 55 summarizes the generation, compilation, and training process. Shading indicates manual creation or markup. Striped boxes are dependent upon the configured input/output methods (i.e., client interface and database).

  1. The ChoiceMaker schema is written. It defines the types of the records to match, the way data are exchanged with the client and the database, and the fields to display in the human review tool.
  2. The generators generate the base holder classes, translators, and human review tool.
  3. The clue set is written. The clue set defines how records are compared.
  4. The ClueMaker compiler is used to create the clue holder classes, which are subclasses of the base holder classes, and the clue evaluation class.
  5. A set of record pairs that are potential matches are extracted from the customer's database.
  6. Each record pair is marked by clerks as `match', `differ', or `hold' using the generated human review tool. Alternatively, a hand-written tool may be used.
  7. The ChoiceMaker training application completes the probability model by removing irrelevant clues from the clue set and determining weights for the clues that we will be used in production.
  8. The probability model is tested with the ChoiceMaker training application.
