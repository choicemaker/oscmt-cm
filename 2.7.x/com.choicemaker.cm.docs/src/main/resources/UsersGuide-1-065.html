2.1 Working directory

The working directory is specified with the workingDir element. Relative paths are relative to the location of the configuration file. The default is `.', i.e., the directory in which the configuration file resides.

2.2 Classpath

The classpath elements are used to extend the classpath on which Java looks for classes referenced by the schema, ClueMaker files, plugins, and classes loaded by them. The content of the classpath elements is in the standard classpath format. Colons and semicolons may be used as path separators on all platforms. Directories must have a trailing forward slash.

There may be two classpath elements, one directly under core and one under reload. Classes that are on the system classpath and/or on the outer classpath element, are loaded only once by ModelMaker. Classes that are not on the two previous classpaths and are on the reload classpath are reloaded each time a new model is loaded or a model is rebuilt or reloaded in ModelMaker. Hence, classes that may take long to initialize and are not modified, should be on the system or outer classpath. Classes that are modified should be on the reload classpath so that they can be modified, compiled, and tested in ModelMaker without exiting and restarting ModelMaker.

For ChoiceMaker servers, the distinction between the two classpaths is irrelevant. All classes are loaded only once when they are first used, even if there are multiple production probability models that reference them.

To enhance portability, it is suggested that classpaths in configuration files and plugins are relative.
