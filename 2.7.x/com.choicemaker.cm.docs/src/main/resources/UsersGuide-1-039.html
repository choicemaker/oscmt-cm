5 Validity predicates

Each field of the schema has a validity predicate defined in form of a Boolean Java expression. For example, the validity of firstName is defined to be StringUtils.nonEmptyString(firstName) on line 6. StringUtils is a class in the package com.choicemaker.cm.matching.gen imported on line 3 and nonEmptyString is a static method of this class.

If the validity attribute is omitted, it defaults to true.

The encoding of `&' by `&amp;' and `"' by `&quot;', as shown in the validity predicate of lastName, is mandated by XML. The user may choose to edit schemas with an XML editor that makes this encoding transparent.
