11 Filters

Filters can be used to select a subset of a marked record pair source for displaying in the human review panel. Filtering can be performed based on the human decision, ChoiceMaker's decision, ChoiceMaker's probability, and the clue firings/values as show in Figure 25.

Additionally, the filter allows for undersampling of records in certain probability ranges. This is, for example, useful for selecting a subset from a generated set of record pairs aimed for human markup. Undersampling is defined by the argument of Limited by. For example, `0-1:5:100' only lets records with a match probability between 0 and 1% through the filter with a probability of 5% and limits the maximum number of records in the probability range to 100. In general, the format of the argument is given by the following EBNF:

LIMITED ::= LIMITATION {`,' LIMITATION }

LIMITATION ::= FROM-PROBABILITY `-` TO-PROBABILITY `:' PASS-THROUGH-PROBABILITY

`:' MAX-COUNT

The filter can also be based on the checked indices.

Upon setting the filter, the indices of the matching pairs are displayed in the pair list.

All operations on the Train and Test panel that select a subset of the pairs actually set the filter. E.g., our earlier example of clicking into the misses column of clue 3 creates the filter displayed in Figure 25.

Sometimes it is useful to write a ClueMaker rule with decision none to help in selecting a specific subset of pairs.

The pairs that match the current filter's criteria can be saved using Source->Save Selection As.... The user can save data in any format that supports saving, including XML and flat file.

Filtering does not affect the indices of pairs. Pairs that do not match the active filter can still be displayed in the review panel by entering the index or dragging the global slider.

The filter has no effect on the Train and Test panels. To view the values of these panels on a selection, create a new source corresponding to the selection (see above) and then use this source.
