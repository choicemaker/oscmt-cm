4 ChoiceMaker schema

ChoiceMaker uses an XML-based schema, similar to a database schema, to define the type and structure of the entities to be matched. Through extensions, the schema also defines many other aspects:

  • The set of logically valid values for a field, e.g., ruling out `N/A' and `UNKNOWN'.
  • Derived fields, such as the house number parsed from the address.
  • The input and output formats, e.g., the width and order of fields in flat files and the names of tables in Oracle.
  • The fields to be stored for reporting.
  • The fields to be displayed in ModelMaker's review screen.

ChoiceMaker schemas are described in detail in Chapter III.
