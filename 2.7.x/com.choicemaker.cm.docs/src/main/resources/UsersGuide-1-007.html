2 Blocking

Blocking is the first pass of record matching. Its goal is to quickly find all the records that match the query record and at the same time not return too many records. Records found in blocking are passed on to scoring; records that are not identified as potential matches in blocking are not looked at in more detail.

ChoiceMaker's automated blocking is easy to configure and supports a variety of database management systems, including Oracle and Microsoft SQL Server.

The configuration of blocking only requires a selection of blocking fields, such as first name or stock ticker symbol, and the creation of indexes on these fields. Everything else is done automatically by ChoiceMaker's automated blocking. In particular, there is no need to worry about frequencies, e.g., there could be 10,000 Jim's in the database so that first name is not always a suitable blocking field by itself because it may return too many records for scoring.

Blocking handles stacked data, that is multiple values for a certain field. For example, a person might be represented in a database by a row in a Person table and by any number of rows in an Address table. ChoiceMaker's blocking considers all the address rows linked to the same person at once and returns the single Person row together with all the Address rows.

ChoiceMaker's blocking is described in detail in Chapter IX.7.
